# Video Prompt Engineer

> 从剧本到视频提示词，一个 skill 搞定。

一个专为 AI 短剧/视频创作设计的提示词工程 skill。把剧本或小说直接转成可执行的视频生成提示词表格，支持 Seedance、Runway、Pika 等主流视频模型。

---

## 一句话介绍

你丢进来一个剧本，它吐出来一张分镜表。每个镜头该拍什么、说什么、拍几秒，全部写清楚。视频模型按表执行就行。

---

## 核心能力

| 能力 | 说明 |
|------|------|
| **剧本解析** | 自动识别角色、场景、台词，标注参考图位置 |
| **导演决策** | 情绪节拍拆解、叙事编排、空间调度 |
| **镜头切分** | 按台词时长和情绪转折自动切分/合并镜头 |
| **提示词撰写** | 输出带时间戳、景别、动作描述、台词的完整提示词 |
| **质检体系** | 11 项检查确保提示词可执行 |

## 快速开始

### 1. 准备剧本

把剧本保存为 `.txt` 或 `.md` 格式。支持标准剧本格式：

```
01-1 百灵阁荷池，夜，内
人物：苏锦，狐妖小容，沈郎
△珠帘摇曳，莲花飘动...
小容：求阁主今日能遂我心愿...
```

### 2. 调用 Skill

对你的 Agent 说：

```
帮我出一下这个剧本的视频分镜提示词
```

或直接上传剧本文件：

```
@剧本.txt 帮我转成分镜提示词
```

### 3. 确认配置

Skill 会询问三个配置项：

| 配置项 | 示例 |
|--------|------|
| 画幅 | 16:9 或 9:16 |
| 模型 | Seedance 2.0 / Runway / Pika |
| 风格 | 古风短剧，历史剧质感 |

确认后固定，同项目不再重复询问。

### 4. 获取输出

输出为 `.md` 文件，包含：

- **项目配置**：画幅、模型、风格
- **参考图清单**：角色/场景 + 需标注 `@` 的元素
- **空间调度**：无参考图场景的文字推演
- **分镜表格**：镜号 | 提示词 | 时长

## 核心原则

1. **台词即动作**：台词直接写进提示词，时长是刚性约束
2. **情绪物理化**：抽象情绪翻译为物理动作（握拳、眼神、身体距离）
3. **信息宁精勿滥**：每个镜头元素简洁，信息过载比不足更容易翻车
4. **镜头独立性**：每个镜头独立可执行，无跨镜头引用
5. **场景描述保留**：剧本先交代的场景描述不得省略

## 六步执行流程

```
Step 0: 项目配置（画幅/模型/风格）
  ↓
Step 1: 剧本分析（角色/场景/剧情/情绪弧线）
  ↓
Step 2: 导演决策（叙事编排/情绪节拍/空间调度）
  ↓
Step 3: 镜头切分（按台词时长+情绪转折切分/合并）
  ↓
Step 4: 提示词撰写（风格+时间戳分镜+台词+后缀）
  ↓
Step 5: 质检（11项检查，不通过则返工）
```

---

## 质检体系（11项）

| 编号 | 检查项 | 说明 |
|------|--------|------|
| D0 | 台词完整性 | 每段台词完整保留，不可拆分 |
| D1 | 台词可执行性 | 镜头时长 ≥ 台词时长 |
| D2 | 矛盾性 | 无时间/视角/情绪矛盾 |
| D3 | 歧义性 | 模糊表述=随机，必须具体化 |
| D4 | 具象性 | 每句话都能被画出来 |
| D5 | 镜头独立性 | 无跨镜头引用 |
| D6 | 精简性 | 单条 ≤ 300字，无废话 |
| D7 | 动作描述完整性 | 主语+位置，特写除外 |
| D8 | 场景描述保留 | 先交代的场景描述不得省略 |
| D9 | 时间戳格式 | 整数秒，分段写法 |
| D10 | 导演思维 | 元素组合恰当，空间关系一致 |

## 进阶用法

### 剧本解析器

长剧本推荐先用解析器结构化：

```bash
cd video-prompt-engineer/scripts
python3 parser.py script.txt > structured_data.json
```

输出包含角色列表、场景列表、台词列表（含情绪、时长估算）。

### 自定义风格

在 Step 0 中指定风格，如：
- `赛博朋克，霓虹灯质感`
- `日系治愈，柔光滤镜`
- `悬疑惊悚，低饱和高对比`

### 参考图对齐

输出中的 `@` 标记由用户在网页端手动对齐参考图。AI 不需要知道它对应哪张图。
