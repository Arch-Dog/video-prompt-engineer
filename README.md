# Video Prompt Engineer

> 从剧本到视频提示词，一个 skill 搞定。

一个专为 AI 短剧/视频创作设计的提示词工程 skill。把剧本或小说直接转成可执行的视频生成提示词表格，支持 Seedance、Runway、Pika 等主流视频模型。

---

## 一句话介绍

你丢进来一个剧本，它吐出来一张分镜表。每个镜头该拍什么、说什么、拍几秒，全部写清楚。视频模型按表执行就行。

---

## 核心能力

| 能力 | 说明 |
|------|------|
| **剧本解析** | 自动识别角色、场景、台词/旁白/内心OS，标注参考图位置 |
| **导演决策** | 情绪节拍拆解、叙事编排、空间调度、情绪氛围词 |
| **镜头切分** | 按音频时长和情绪转折自动切分/合并镜头，含漏项预防 |
| **提示词撰写** | 输出带时间戳、动作描述、音频内容（台词+旁白+OS）、音色参数的完整提示词 |
| **快速自检** | 3 项快速检查，在提交前扫一遍明显问题 |
| **质检体系** | 11 项分级检查（P0 致命 / P1 严重 / P2 一般），含失败回溯规则 |

## 快速开始

### 1. 准备剧本

把剧本保存为 `.txt` 或 `.md` 格式。支持标准剧本格式：

```
01-1 百灵阁荷池，夜，内
人物：苏锦，狐妖小容，沈郎
△珠帘摇曳，莲花飘动...
小容：求阁主今日能遂我心愿...
```

### 2. 调用 Skill

对你的 Agent 说：

```
帮我出一下这个剧本的视频分镜提示词
```

或直接上传剧本文件：

```
@剧本.txt 帮我转成分镜提示词
```

### 3. 确认配置

Skill 会询问三个配置项：

| 配置项 | 示例 |
|--------|------|
| 画幅 | 16:9 或 9:16 |
| 模型 | Seedance 2.0 / Runway / Pika |
| 风格 | 古风短剧，历史剧质感 |

确认后固定，同项目不再重复询问。

### 4. 获取输出

输出为 `.md` 文件，包含：

- **项目配置**：画幅、模型、风格
- **参考图清单**：角色/场景 + 需标注 `@` 的元素
- **空间调度**：无参考图场景的文字推演 + 衔接手法记录
- **分镜表格**：镜号 | 提示词 | 时长

## 核心原则

1. **音频内容即动作**：台词、旁白、内心OS 统一视为音频内容，时长是刚性约束
2. **你是导演**：群戏给谁特写、节奏快慢、闪回处理、省略保留，由你决定
3. **情绪物理化**：抽象情绪翻译为物理动作（握拳、眼神、身体距离）
4. **画面感优先**：动作描述包含环境互动、质感细节、动态元素
5. **叙事转译**：所有描述必须是摄影机可拍摄的物理现象
6. **镜头独立性**：每个镜头独立可执行，无跨镜头引用
7. **场景描述保留**：剧本先交代的场景描述不得省略
8. **提示词不重复场景前缀**：场景信息通过动作描述自然融入
9. **外貌参考图**：人物外貌不写，参考图已包含

## 七步执行流程

```
Step 0: 项目配置（画幅/模型/风格）
  ↓
Step 1: 剧本分析（角色/场景/剧情/情绪弧线/参考图确认）
  ↓
Step 2: 导演决策（叙事编排/情绪节拍/空间调度/情绪氛围词）
  ↓
Step 3: 镜头切分（按音频时长+情绪转折切分/合并/漏项预防）
  ↓
Step 4: 提示词撰写（情绪氛围+时间戳分镜+音频内容+音色参数）
  ↓
Step 4.5: 快速自检（抽象词/空间追踪/衔接手法）
  ↓
Step 5: 质检（11项分级检查，不通过则返工）
```

---

## 质检体系（11项，P0→P2）

| 编号 | 检查项 | 优先级 | 说明 |
|------|--------|--------|------|
| D0 | 音频完整性 | **P0 致命** | 每段音频完整保留，不可拆分；无遗漏 |
| D1 | 音频可执行性 | **P0 致命** | 镜头时长 ≥ 音频总时长 |
| D4 | 画面感 | **P1 严重** | 动作描述包含环境互动/质感细节/动态元素 |
| D5 | 镜头独立性 | **P1 严重** | 无跨镜头引用 |
| D7 | 动作描述完整性 | **P1 严重** | 主语+位置+面部朝向/视线 |
| D10 | 镜头合并验证 | **P1 严重** | 空间关系/视线/180度规则/时间连贯性 |
| D11 | 提示词空间验证 | **P1 严重** | Exit/Entry Frame 共享视觉锚点 |
| D12 | 音频数量核对 | **P1 严重** | 剧本音频总数 == 分镜表音频总数 |
| D2/D3 | 一般性检查 | P2 一般 | 无时间/视角/情绪矛盾；无歧义 |
| D6 | 信息密度 | P2 一般 | 3-5秒一个视觉事件，单条≤300字 |
| D8 | 场景描述保留 | P2 一般 | 先交代的场景描述不得省略 |

**失败回溯**：质检不通过则按规则回退到对应步骤修改，连续两次失败返回上一级重新审视。

## 进阶用法

### 剧本解析器

长剧本（台词≥20句或场景≥5个）推荐先用解析器结构化：

```bash
cd video-prompt-engineer/scripts
python3 parser.py script.txt > structured_data.json
```

输出包含角色列表、场景列表、台词列表（含情绪、时长估算）。

### 自定义风格

在 Step 0 中指定风格，如：
- `赛博朋克，霓虹灯质感`
- `日系治愈，柔光滤镜`
- `悬疑惊悚，低饱和高对比`

### 参考图对齐

输出中的 `@` 标记由用户在网页端手动对齐参考图。AI 不需要知道它对应哪张图。
